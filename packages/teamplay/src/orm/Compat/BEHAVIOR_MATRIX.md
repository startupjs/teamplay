# Поведенческая матрица compat / non-compat по тестам

Документ фиксирует более детальную разницу поведения, извлечённую из тестов и текущего кода. Это не список рекомендаций.

Основные источники:

| Источник | Что покрывает |
|---|---|
| `test/signalCompat.js` | Compat API, Racer-like overloads, refs, query helpers, start/stop, model events. |
| `test/sub$.js` | Базовое поведение `sub()` и public documents. |
| `test/$.js` | Базовое поведение local/private signals, `assign`, local mutators. |
| `test/setDiffDeep.js` | Низкоуровневая утилита `setDiffDeep`. |
| `test/compatNullishSemantics.js` | Null/undefined/sparse-array паритет local/public writes. |
| `test/idFields.js` | `_id`/`id` в base mode. |
| `test/publicOnlyCompat.js` | Deprecated `publicOnly` no-op and global-root private write warning. |
| `test/compatBatchReadiness.js` | Readiness query/aggregation для compat batch hooks. |
| `test/subscriptionManagers.js` | Subscription transport, GC delay, materialization, query params. |

Варианты миграции и ограничения целевого non-compat API фиксируются в `src/orm/Compat/MIGRATION_OPTIONS.md`.

Decision-ready рекомендации по этим различиям вынесены в `src/orm/Compat/MISALIGNMENTS.md`, а практические before/after примеры - в `src/orm/Compat/REFACTORING_EXAMPLES.md`.

## Коротко про `set`

Самая важная неоднозначность: слово `set` встречается на разных слоях.

| Слой | Фактическое поведение |
|---|---|
| Low-level `dataTree.set()` | Для object/array значений обновляет текущий node через `setDiffDeep()` внутри, если это безопасно. Это сделано для сохранения реактивности child signals, а не для публичной merge-семантики. Источник: `src/orm/dataTree.js:99`, `src/utils/setDiffDeep.js:13`, `test/setDiffDeep.js:6`. |
| Base `Signal.set(value)` | Принимает один аргумент. Для private/local paths идёт в `setPrivateData()` -> `dataTree.set()`, то есть object/array могут обновляться in-place через внутренний deep diff. Источник: `src/orm/SignalBase.ts:370`, `src/orm/signalValueMutations.ts:30`, `src/orm/privateData.js:35`. |
| Base public doc `.set(value)` | Пишет через `setPublicDoc()`. Whole-doc write заменяет документ относительно переданного объекта: старые поля, которых нет в новом объекте, удаляются. Источник: `test/sub$.js:128`, `src/orm/dataTree.js:228`. |
| Base public subpath `.set(value)` | Пишет subpath через clone old doc -> `dataTree.set()` -> json diff. Для object value это ведёт себя как замена target object: stale keys на target path удаляются внутренним `setDiffDeep`. Источник: `src/orm/dataTree.js:236`. |
| Compat `SignalCompat.set(path?, value)` | Для non-`undefined` использует replace-путь `setReplaceOnSignal()`: private -> `setReplacePrivateData()`, public -> `setPublicDocReplace()`. То есть compat `set` на target path является replace, не deep merge. Источник: `src/orm/Compat/SignalCompat.js:204`, `src/orm/Compat/SignalCompat.js:1152`, `test/signalCompat.js:768`, `test/signalCompat.js:1484`. |
| Compat `SignalCompat.set(undefined)` | Специально уходит в base `Signal.prototype.set()` для сохранения Racer-like nullish semantics. Local/private сохраняет explicit `undefined`; public нормализует `undefined` в `null`. Источник: `src/orm/Compat/SignalCompat.js:215`, `test/signalCompat.js:728`, `test/signalCompat.js:3133`, `test/compatNullishSemantics.js:91`. |
| Compat `SignalCompat.setDiffDeep(path?, value)` | Отдельный API: рекурсивно diff-ит object/array, удаляет stale keys, старается сохранить child signals и атомарность scheduled observers. Это не то же самое, что compat `set`. Источник: `src/orm/Compat/SignalCompat.js:275`, `src/orm/Compat/SignalCompat.js:961`, `test/signalCompat.js:823`, `test/signalCompat.js:1027`. |

|---|---|---|---|
| Сигнатура `set` | `set(value)` и `set(path, value)`. | `set` на public object subpath | Direct replace json0 op, если ancestors существуют: `{ p: ['profile'], od: old, oi: new }`. | `set` object internals | Compat `set` не использует `dataTree.setDiffDeep` для обычного non-`undefined` local write; он идёт через `setReplacePrivateData()`. | Base private/local `set` идёт через `setPrivateData()` и низкоуровневый `dataTree.set()`, где object/array обновляются через `setDiffDeep()`. | `src/orm/Compat/SignalCompat.js:1152`, `src/orm/privateData.js:35`, `src/orm/dataTree.js:99`. |

## Выводы без рекомендаций

| Наблюдение | Деталь |
|---|---|
| `set` нельзя описать одной фразой | Base local/private set использует internal deep diff для object/array. Compat `set` для non-`undefined` сознательно идёт через replace path. Public writes используют ShareDB/json0 и могут выглядеть как diff ops, но user-facing target semantics остаётся replacement, а не merge. |
| `setDiffDeep` в compat является отдельным legacy API | Он не равен обычному compat `set`. Он нужен для recursive stale-key deletion, array diff и сохранения child-signal reactivity under observers/start/refs. |
| `setEach` не эквивалентен base `assign` | В compat `setEach` ставит `null`/`undefined` как значения по Racer semantics. В base `assign` `null`/`undefined` удаляют поля. |
| Public nullish semantics отличаются от local | Local explicit `undefined` сохраняется как `undefined`; public remote writes нормализуют `undefined` в `null`. Это покрыто отдельными tests и важно для sparse arrays/started targets. |
| Значительная часть отличий не в наличии метода, а в overload/readiness/lifecycle semantics | Например `subscribe/fetch`, `query.getExtra`, refs, `start/stop`, model events и hooks могут иметь похожие низкоуровневые primitives, но compat tests фиксируют дополнительные Racer-like гарантии. |
